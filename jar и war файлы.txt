Файлы JAR и WAR в Java служат для упаковки и распространения Java-приложений, но используются в разных контекстах:
?? JAR (Java ARchive)

    Назначение: Упаковка обычных Java-программ или библиотек.

    Содержимое:

        .class-файлы (скомпилированные классы)

        META-INF/MANIFEST.MF (манифест с основной точкой входа Main-Class, если это исполняемый JAR)

        Ресурсы (иконки, конфиги и т. д.)

    Пример использования:

        Библиотека (library.jar)

        Консольное Java-приложение (myapp.jar)

?? WAR (Web ARchive)

    Назначение: Упаковка веб-приложений для развертывания на Java EE-серверах (например, Tomcat, WildFly).

    Содержимое:

        Структура каталогов для веб-приложений:

        /WEB-INF/
            web.xml            (дескриптор развертывания)
            classes/           (классы и ресурсы)
            lib/               (JAR-файлы библиотек)
        index.jsp             (или другие веб-страницы)

        JSP-файлы, HTML, CSS, JS и т.д.

    Пример использования:

        Веб-приложение online-shop.war, которое можно задеплоить на сервер.

?? Краткое сравнение
Характеристика	JAR	WAR
Тип приложения	Библиотека / консольное ПО	Веб-приложение (Servlet/JSP)
Точка входа	Main-Class в MANIFEST.MF	Через web.xml или аннотации
Развёртывается на	Любая JVM	Java EE-сервер (например, Tomcat)
Статические ресурсы	Может содержать	Явно структурированы (JSP, HTML и т.д.)
