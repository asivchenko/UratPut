
JVM (Java Virtual Machine, виртуальная машина Java) — это программа, которая исполняет Java-программы.
Она является ключевым компонентом платформы Java.

Вот суть понятия:
🏗️ Что делает JVM

✅ Запускает Java-программы — она читает байт-код .class (скомпилированный код Java) и исполняет его.

✅ Обеспечивает кроссплатформенность — одна и та же .class или .jar программа может работать на Windows, Linux, Mac и т. д., если на устройстве установлена JVM.

✅ Управляет памятью — автоматический сборщик мусора (GC), управление кучей (Heap), стеком и т.п.

✅ Обеспечивает безопасность — изолированное выполнение, проверка байт-кода.

✅ Оптимизирует выполнение — JIT-компиляция (Just-In-Time) ускоряет работу, компилируя горячие участки байт-кода в машинный код.
⚙️ Как работает JVM (упрощённо)

    Ты пишешь программу на Java → HelloWorld.java

    Компилируешь → javac HelloWorld.java → получаешь HelloWorld.class (байт-код)

    Запускаешь → java HelloWorld

    JVM:

        читает HelloWorld.class

        интерпретирует байт-код или компилирует его в машинный код

        выполняет программу

🖼️ Структура платформы Java

Твой код на Java (.java)
            ↓
      Java Compiler (javac)
            ↓
       Байт-код (.class)
            ↓
          JVM
      (исполняет байт-код)
            ↓
      Машинный код (CPU)

Итог

    JVM позволяет запускать Java-программы на разных ОС без перекомпиляции.

    Защищает память, управляет потоками, оптимизирует производительность.

    Вместе с JDK и JRE формирует Java-платформу.
