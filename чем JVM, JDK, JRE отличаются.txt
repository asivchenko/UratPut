
🧠 1. JVM (Java Virtual Machine) — исполнитель

    Это виртуальная машина, которая исполняет байт-код (.class).

    Платформозависимая программа: для Windows — одна, для Linux — другая.

    Отвечает за:

        выполнение Java-кода,

        сборку мусора (GC),

        управление памятью,

        JIT-компиляцию.

🔧 JVM — часть JRE и JDK.
📦 2. JRE (Java Runtime Environment) — среда выполнения

    Это набор компонентов, необходимых для запуска Java-программ.

    Содержит:

        JVM

        библиотеки классов Java (rt.jar и др.)

        другие runtime-инструменты

💡 JRE нужен, если ты хочешь только запускать Java-программы, но не писать и не компилировать их.
🧰 3. JDK (Java Development Kit) — набор для разработчика

    Это полный комплект для разработки Java-программ.

    Содержит:

        JRE (а значит, и JVM)

        компилятор javac (преобразует .java в .class)

        отладчики, утилиты (javadoc, javap, jar и др.)

💡 JDK нужен, если ты хочешь писать, компилировать и запускать Java-программы.
